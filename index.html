The Problem

Bitcoin's cryptography has an expiry date.

The Solution

Meet BIP 360

Pay-to-Merkle-Root (P2MR) — a new output type that gives Bitcoin quantum resistance without breaking anything.

Merkle Root Commitment

Outputs commit to a Merkle root of spending conditions instead of exposing a public key directly.

Taproot Compatible

Same functionality as Taproot — but with the quantum-vulnerable keypath spending removed.

Tapscript Native

Fully tapscript-native address type, protected from long-exposure quantum attacks.

Soft Fork Ready

Can be deployed via soft fork with full backward compatibility. No chain disruption.

How P2MR Works

Instead of committing to a public key (which a quantum computer could reverse), P2MR commits to a Merkle root — a hash of all valid spending scripts.

The public key is never exposed until the moment of spending, and by then, the transaction is already confirmed.

Output: OP_1 <merkle_root> Spend: <script> <proof> <witness>
